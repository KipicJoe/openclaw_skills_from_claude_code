# OpenClaw 技能学习指南

从 Claude Code 学习的7个高级技能的完整学习指南。

## 学习目标

完成本指南后，你将能够：
1. 理解每个技能的核心概念和设计思想
2. 掌握技能的使用方法和最佳实践
3. 了解技能间的协同工作方式
4. 将技能应用到实际任务中
5. 根据需要定制和扩展技能

## 学习路径

### 阶段一：基础技能（第1-2周）
**目标**: 掌握最实用、最常用的技能

#### 1. 记忆审查技能 (`memory-review`)
- **学习重点**: 记忆层概念、审查流程、用户批准机制
- **实践任务**: 运行一次完整的记忆审查，批准至少3个变更
- **掌握标准**: 能解释记忆层的区别和审查原则

#### 2. 定时任务技能 (`kairos-lite`)
- **学习重点**: 自然语言解析、cron转换、任务管理
- **实践任务**: 创建3个不同间隔的定时任务并管理它们
- **掌握标准**: 能创建和管理各种定时任务

### 阶段二：质量技能（第3-4周）
**目标**: 掌握提高任务质量的技能

#### 3. 验证门技能 (`verification-gate`)
- **学习重点**: 验证类型、验证流程、安全机制
- **实践任务**: 为不同类型的任务设置验证检查
- **掌握标准**: 能为常见任务设计适当的验证方案

#### 4. 上下文压缩技能 (`context-compressor`)
- **学习重点**: 9段式结构、压缩策略、质量保证
- **实践任务**: 压缩一个长会话并评估信息保留情况
- **掌握标准**: 能有效压缩上下文而不丢失关键信息

### 阶段三：高级技能（第5-6周）
**目标**: 掌握智能化和自动化技能

#### 5. 风格提取技能 (`style-extractor`)
- **学习重点**: 模式识别、偏好学习、个性化适应
- **实践任务**: 分析自己的交互历史，提取3个偏好模式
- **掌握标准**: 能识别和适应用户的交互风格

#### 6. 梦想记忆技能 (`dream-memory`)
- **学习重点**: 自动整理、触发机制、优化策略
- **实践任务**: 配置并运行自动记忆整理
- **掌握标准**: 能配置和维护自动记忆整理系统

### 阶段四：专家技能（第7-8周）
**目标**: 掌握复杂任务处理技能

#### 7. 群组协调技能 (`swarm-coordinator`)
- **学习重点**: 任务分解、代理协调、结果合成
- **实践任务**: 协调多个代理完成一个复杂任务
- **掌握标准**: 能有效分解和协调多代理任务

## 技能详细学习指南

### 技能1: 记忆审查 (`memory-review`)

#### 核心概念
- **记忆层**: OpenClaw 的多层记忆系统
- **审查流程**: 收集→分析→提议→批准→执行
- **安全原则**: 始终征求用户批准，保留历史

#### 学习步骤
1. **了解记忆结构**
   - 阅读 `MEMORY.md`、`memory/` 目录结构
   - 理解不同记忆层的用途

2. **运行首次审查**
   ```
   /memory-review --dry-run
   ```
   - 观察分析过程
   - 查看提出的变更建议

3. **执行实际审查**
   ```
   /memory-review
   ```
   - 批准合理的变更
   - 拒绝不确定的变更
   - 观察执行结果

4. **学习高级功能**
   - 尝试不同的选项: `--recent-days`, `--full-scan`
   - 理解分类规则和目的地选择

#### 实践任务
- 任务1: 审查最近3天的记忆
- 任务2: 识别并解决一个记忆冲突
- 任务3: 将临时记忆提升为长期记忆

### 技能2: 定时任务 (`kairos-lite`)

#### 核心概念
- **自然语言解析**: 从自然语言中提取间隔和时间
- **cron转换**: 将间隔转换为标准的cron表达式
- **任务管理**: 创建、查看、管理定时任务

#### 学习步骤
1. **创建简单定时任务**
   ```
   /loop 5m check system status
   /loop 1h backup important files
   ```

2. **管理定时任务**
   ```
   /loop list
   /loop status
   /loop cancel <task-name>
   ```

3. **学习高级调度**
   ```
   /loop daily at 9 AM send report
   /loop weekdays at 18:00 end of day summary
   ```

4. **集成其他技能**
   ```
   /loop 1h /memory-review --recent-days 1
   /loop daily at 23:00 /system-cleanup
   ```

#### 实践任务
- 任务1: 创建每分钟、每小时、每天的任务各一个
- 任务2: 创建基于时间的任务（特定时间点）
- 任务3: 将其他技能集成到定时任务中

### 技能3: 验证门 (`verification-gate`)

#### 核心概念
- **验证类型**: 文件、代码、系统、功能、安全验证
- **验证流程**: 定义→执行→分析→行动
- **安全机制**: 失败处理、回滚、通知

#### 学习步骤
1. **基本文件验证**
   ```
   /verify file:app.py --syntax
   /verify file:config.yaml --valid
   ```

2. **系统状态验证**
   ```
   /verify "systemctl status nginx"
   /verify "curl -f http://localhost:8080/health"
   ```

3. **任务后验证**
   - 完成一个任务后立即运行验证
   - 观察验证结果和后续行动

4. **自定义验证脚本**
   - 创建简单的验证脚本
   - 集成到验证流程中

#### 实践任务
- 任务1: 为文件编辑设置语法验证
- 任务2: 为服务重启设置状态验证
- 任务3: 创建自定义验证检查

### 技能4: 上下文压缩 (`context-compressor`)

#### 核心概念
- **9段式结构**: 系统化的信息组织方式
- **压缩策略**: 语义、层次、时间、相关性压缩
- **质量保证**: 信息保留验证、准确性检查

#### 学习步骤
1. **基本压缩**
   ```
   /compress conversation
   /compress memory
   ```

2. **查看压缩结果**
   - 分析9个段落的组织
   - 评估信息保留情况
   - 检查技术准确性

3. **定制压缩**
   ```
   /compress --segments 1,3,6,9
   /compress --ratio 0.3
   /compress --preserve-code
   ```

4. **恢复和扩展**
   - 从压缩上下文恢复工作
   - 验证压缩不影响任务连续性

#### 实践任务
- 任务1: 压缩一个长会话并评估效果
- 任务2: 尝试不同的压缩设置
- 任务3: 从压缩上下文继续工作

### 技能5: 风格提取 (`style-extractor`)

#### 核心概念
- **模式识别**: 从交互中识别重复模式
- **偏好学习**: 学习用户的偏好和习惯
- **个性化适应**: 根据学习结果调整行为

#### 学习步骤
1. **初始分析**
   ```
   /style-extractor analyze --days 7
   ```
   - 查看发现的模式
   - 理解置信度评分

2. **模式验证**
   - 审查提取的模式
   - 确认或纠正模式
   - 更新用户配置文件

3. **观察适应效果**
   - 注意技能如何适应你的风格
   - 提供反馈以改进学习

4. **高级配置**
   - 调整提取敏感度
   - 设置学习优先级
   - 管理学习数据

#### 实践任务
- 任务1: 分析自己的通信风格偏好
- 任务2: 识别工作流程偏好
- 任务3: 验证并完善提取的模式

### 技能6: 梦想记忆 (`dream-memory`)

#### 核心概念
- **自动整理**: 后台自动运行的内存优化
- **触发机制**: 时间、会话、活动等多重触发
- **优化策略**: 提升、压缩、组织、清理

#### 学习步骤
1. **初始配置**
   - 查看默认配置
   - 理解各触发条件
   - 设置适合的频率

2. **观察运行**
   - 等待自动运行或手动触发
   - 查看运行日志和报告
   - 理解执行的操作

3. **效果评估**
   - 比较整理前后的内存状态
   - 评估整理效果
   - 调整配置参数

4. **集成维护**
   - 与其他技能协同工作
   - 建立定期维护流程
   - 监控系统健康

#### 实践任务
- 任务1: 配置并触发一次整理
- 任务2: 分析整理报告和效果
- 任务3: 建立定期整理计划

### 技能7: 群组协调 (`swarm-coordinator`)

#### 核心概念
- **任务分解**: 将复杂任务分解为可并行部分
- **代理协调**: 管理多个代理的协作
- **结果合成**: 整合各代理的结果

#### 学习步骤
1. **简单协调**
   ```
   /swarm "简单的多步骤任务"
   ```
   - 观察任务分解
   - 查看代理分配
   - 跟踪执行进度

2. **复杂任务协调**
   - 尝试更复杂的任务
   - 观察更精细的分解
   - 学习冲突解决

3. **协调模式**
   - 理解不同的协调模式
   - 尝试不同的代理组合
   - 学习结果整合策略

4. **优化和调整**
   - 根据结果调整策略
   - 优化资源分配
   - 改进协调效率

#### 实践任务
- 任务1: 协调一个中等复杂度的任务
- 任务2: 观察并理解协调过程
- 任务3: 优化协调策略

## 技能协同学习

### 协同模式1: 完整工作流
```
用户请求 → [群组协调]分解 → [多个代理执行] → 
[验证门]验证 → [上下文压缩]优化 → 
[梦想记忆]整理 → [记忆审查]用户检查
```

### 协同模式2: 学习优化循环
```
[风格提取]学习模式 → [自我调整] → 
[用户交互]产生新数据 → [梦想记忆]整理 → 
[风格提取]继续学习
```

### 协同模式3: 定期维护
```
[定时任务]调度 → [记忆审查]运行 → 
[梦想记忆]整理 → [验证门]检查 → 
[报告生成]通知用户
```
